# Bloque I — Introducción

## Objetivo del Paso 17

<small>§2</small>

En este paso la IA **construye el sitio web funcional** ejecutando — sin decidir — los outputs de Pasos 1-16. Stack canónico: Astro 5 + Tailwind v3 + pnpm + Cloudflare Pages. **7 fases operativas** (Fase 0 Acta de Lectura + Fase 1 Setup + Fase 2 UI Components + Fase 3 Layout Global + Fase 4 Construcción de Páginas + Fase 5 SEO/Schema + Fase 6 Limpieza de Rastros IA) con **2 paradas de confirmación** (PARADA 1 después de Fase 0, PARADA 2 antes del Deploy). El Paso 17 es **ejecutor, no decisor**: cualquier contradicción detectada para hasta resolución upstream.

**Outputs del paso:**

- **17.1** Acta de Lectura y Autocontraste — sin contradicciones detectadas en los outputs upstream
- **17.2** Proyecto Astro 5 + Tailwind v3 + pnpm inicializado
- **17.3** 7 componentes UI parametrizados (Button, Card, Badge, Hero, Testimonial, FAQ, ContactForm)
- **17.4** Header + Footer + Layout wrapper global
- **17.5** 28 páginas SEO base + /contacto/ construidas según URL Matrix
- **17.6** Schemas implementados y validados (LocalBusiness, Service, Article, FAQPage, BreadcrumbList, Speakable)
- **17.7** Limpieza de rastros IA (README, comentarios, meta generator, package.json)
- **17.8** Build local pasando sin errores (`pnpm install && pnpm build` ✓)

**Errores que previene:**

- Construir sin Acta de Lectura (contradicciones upstream se propagan)
- Stack heterogéneo (mezclar Astro + React + Vue rompe coherencia)
- Componentes UI inventados sin parametrización (rompe reusabilidad)
- Páginas con stack divergente (rompe SEO + performance)
- Schemas inválidos o incompletos
- Rastros de IA visibles en producción (README "Generated by AI", meta generator, etc.)
- Deploy con build local roto

**Cuándo se ejecuta:** después de Paso 16 (Diseño/Layout) cerrado. Antes de Paso 18 (QA y Deploy).
