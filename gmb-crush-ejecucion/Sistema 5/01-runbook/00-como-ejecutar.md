# Cómo ejecutar el sistema

## Regla de ejecución

```text
No ejecutes un bloque si el gate anterior está BLOCKED.
No inventes datos.
No ocultes placeholders.
No crees GBP con datos en ⚠.
```

---

## Orden completo

```text
G0 — Leer teoría
G1 — Validar preflight
Bloque 1 — Fundamentos
Bloque 2 — Arquitectura
Bloque 3 — Priorización
Bloque 4 — Automatización
Bloque 5 — Salida a Mercado
G4 — Web Live Ready
Bloque 6 — Info que falta
G6 — GBP Safe
Bloque 7 — Crear GBP
G7 — Post-lanzamiento
```

---

## Modo de trabajo de la IA

Para cada bloque:

```text
1. Leer el a-doc de ejecución.
2. Leer los outputs esperados.
3. Ejecutar con datos disponibles.
4. Marcar cada output con estado.
5. Crear/actualizar b-doc o Decision Log.
6. Registrar faltantes.
7. Ejecutar check del bloque.
8. Pedir aprobación si hay 🔁 o ⛔.
```
